# OVN-Kubernetes K8s Rebase Automation - Design Document

## Problem

The OVN-Kubernetes "K8s rebase" bumps all `k8s.io/*` dependencies to a new
Kubernetes version across 3 Go modules. It happens ~3x/year, takes weeks of
human effort, and is ~70% mechanical. The 1.35 rebase (PR #6099) touched 2000+
files and 160k+ lines, but only ~150 lines of actual human-authored code fixes
were needed.

## Scope

- **Target**: upstream repo only (`ovn-org/ovn-kubernetes`)
- **Deliverables**: automation only (no educational/learn skill)
- **Location**: workspace-level command at `~/ovnk/.claude/commands/k8s-rebase.md`
  plus a standalone script at `~/ovnk/ovn-org/ovn-kubernetes/scripts/k8s-rebase.sh`

## Architecture

Two files with a clear split of responsibilities:

### 1. `scripts/k8s-rebase.sh` (~450 lines)

Standalone bash script handling all deterministic work. Can run without
Claude Code. Supports `--dry-run`.

### 2. `.claude/commands/k8s-rebase.md` (~600 lines)

Claude Code skill file. Invokes the script for mechanical phases, then
takes over with agent-guided instructions for build error fixups. Includes
an embedded breakage pattern database.

## The Five Phases

### Phase 0: Validation

- Parse version arg as `X.Y.Z` or `X.Y` (default Z=0)
- Derive versions: `K8S_FULL=v1.36.0`, `API_VERSION=v0.36.0`
- Idempotency: if `go-controller/go.mod` already has target version, exit 0
- Verify target exists on Go module proxy
- Auto-discover controller-runtime version (pattern: `v0.$((K8S_MINOR - 12)).x`)
- Auto-discover Go version from upstream k8s go.mod
- Auto-discover structured-merge-diff major version
- Resume detection: check which commits already exist, skip completed phases

### Phase 1: Module Dependency Updates (3 commits)

Three modules updated in this order (matching PR #6099 convention):

**1a. test/e2e** - 14 `go get` commands + `go mod tidy`
**1b. test/conformance** - 4 `go get` commands + `go mod tidy`
**1c. go-controller** - 16 `go get` commands + `go mod tidy` + `go mod vendor`

Each gets its own commit with body listing all `go get` commands executed.
Commit message format: `"test/e2e rebase"`, `"test/conformance rebase"`,
`"go-controller rebase"`.

#### Key Dependencies per Module

**go-controller** (version-pinned to k8s):

- `k8s.io/api`, `apimachinery`, `apiserver`, `client-go`, `component-helpers`,
  `controller-manager`, `kubelet`, `apiextensions-apiserver` (all `v0.XX.Y`)
- `k8s.io/kubernetes` (`v1.XX.Y`)
- `sigs.k8s.io/controller-runtime` (auto-discovered)
- `sigs.k8s.io/structured-merge-diff/vN`, `knftables`, `json`, `yaml` (latest)
- `k8s.io/utils`, `kube-openapi` (latest)
- `github.com/openshift/api`, `client-go` (latest)

**test/e2e** (version-pinned):

- `k8s.io/kubernetes`, `api`, `client-go`, `pod-security-admission`, `kubectl`
- Plus indirects: `mount-utils`, `cri-client`, `dynamic-resource-allocation`,
  `kube-scheduler`, `csi-translation-lib`, `cloud-provider`, `controller-manager`

**test/conformance** (minimal):

- `k8s.io/apimachinery`, `client-go`
- `sigs.k8s.io/controller-runtime`, `network-policy-api`

### Phase 2: Code Generation (1 commit)

- Update version pin in `go-controller/hack/update-codegen.sh`
  (`k8s.io/code-generator/cmd/%s@v0.XX.Y`)
- Run `make -C go-controller codegen`
- Commit: `"Bump codegen"`

### Phase 3: Version Reference Updates (1 commit)

Pattern-based `sed` replacements (old version as search, not line numbers):

**K8S_VERSION** (6 files):

- `.github/workflows/test.yml` (K8S_VERSION env)
- `.github/workflows/performance-test.yml`
- `contrib/kind-common.sh` (default K8S_VERSION)
- `test/scripts/install-kind.sh`
- `test/scripts/upgrade-ovn.sh`
- `docs/developer-guide/local_testing_guide.md`

**Go version** (only if changed, 6 files):

- `go-controller/Makefile` (GO_VERSION)
- `dist/images/Makefile` (GO_VERSION)
- `.github/workflows/docker.yml` (BUILDER_IMAGE)
- 3 `go.mod` files (`go` + `toolchain` directives)

**Docs** (2 files):

- `docs/features/requirements.md` (version matrix)
- `docs/installation/launching-ovn-kubernetes-on-kind.md`

Commit: `"Updates kubernetes versions"` with body listing files.

### Phase 4: Build Validation and Fixups (agent-guided)

After the script completes, the agent takes over:

1. Run `cd go-controller && go build ./...` -- parse errors
2. Run `cd go-controller && make lint` -- categorize failures
3. Consult the breakage pattern database (10 known patterns)
4. Apply mechanical fixes, present non-mechanical issues to user
5. Create fixup commits matching team convention

## Breakage Pattern Database

10 patterns from analyzing the 1.31-1.35 rebases:

| ID | Pattern | Example (version) |
| ---- | --------- | ------------------- |
| FG-DEFAULT | Feature gate default changed | WatchListClient (1.35) |
| FUNC-RENAME | Deprecated function renamed | WaitForServiceEndpointsNum (1.35) |
| SIG-CHANGE | Function signature changed | IsNodeReady needs logger (1.35) |
| ADMISSION-API | Admission API migration | WithCustomValidator to WithValidator (1.35) |
| EVENT-RECORDER | Event recorder API changed | GetEventRecorderFor (1.35) |
| FAKE-CLIENT | NewSimpleClientset deprecated | SA1019 exclusion needed (1.35) |
| STDLIB-MIGRATE | k8s utility moved to stdlib | slices, context packages (1.35) |
| CONTEXT-LEAK | Context cancel not deferred | lostcancel linter (1.35) |
| VALIDATION-MSG | Error message format changed | test assertion strings (1.35) |
| TIMEOUT-PERF | Slower fake client init | context.WithTimeout ordering (1.35) |

Each pattern in the skill includes: grep detection command, code
transformation recipe, and files typically affected.

## Version Discovery Logic

### controller-runtime

Historical mapping: k8s 1.33 = CR v0.21, k8s 1.34 = CR v0.22, k8s 1.35 = CR
v0.23. Pattern: `v0.$((K8S_MINOR - 12)).x`. Verify via Go module proxy before
using. Fall back to `go list -m -versions` if computed version doesn't exist.

### Go version

Fetch from upstream k8s go.mod:
`curl -s "https://raw.githubusercontent.com/kubernetes/kubernetes/v1.XX.Y/go.mod"`

### structured-merge-diff major version

Also fetched from upstream k8s go.mod (moved from v4 to v6 between 1.34 and
1.35).

## Commit Convention (from PR #6099)

All commits use `--signoff`. The exact sequence:

1. `test/e2e rebase` -- body has go get recipe
2. `test/conformance rebase` -- body has go get recipe
3. `go-controller rebase` -- body has go get recipe
4. `Bump codegen` -- body says what was bumped
5. `Updates kubernetes versions` -- body lists files
6. (optional fixups) `Fix e2e tests failure`, `Fix lint issues`, etc.

## Error Handling

- `go get` failure: print error + Go proxy URL for manual checking
- `go mod tidy` failure: print error + suggest `go get <pkg>@latest`
- Codegen failure: check if controller-gen version needs bumping
- Resume detection: check git log for existing commits, skip completed phases
- `--dry-run` mode: print all commands without executing
