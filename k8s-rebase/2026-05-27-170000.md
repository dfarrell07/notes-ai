# K8s Rebase Skill Testing: 7 Blind Iterations Against ovn-kubernetes

## Context

Tested the k8s-rebase skill (ai-helpers `k8s-rebase-skill` branch) by running
blind agents against ovn-kubernetes upstream (ovn-org/ovn-kubernetes) rebasing
from k8s 1.35 to 1.36. Each iteration reset to clean master and gave a fresh
agent only the skill path -- no hints about expected errors.

## Branch State

- **ai-helpers branch:** `k8s-rebase-skill` with 11 commits on top of main
- **Plugin version:** 0.6.0
- **Files changed:** 7 files, +343/-57 lines

## Key Skill Improvements Made

### Critical fixes (caused blind test failures before)

1. **Step 3b: Mandatory feature gate verification** -- without this, agents
   skipped AtomicFIFO investigation. Tests hung silently and agents reported
   "pass" because the timeout was classified as unrelated. Adding an explicit
   step to test each gate in `new-gates.txt` with a short timeout fixed this.

2. **--userns=keep-id for podman** -- containers without this created
   root-owned files in vendor/ that made the repo unmanageable afterward
   (git checkout, rm all fail with Permission denied). Fixed in both
   k8s-rebase.sh and k8s-rebase-validate.sh.

3. **Gate detection awk fix** -- the original awk only matched constant
   declarations (`Feature = "..."`) but not map keys (`AtomicFIFO: {`).
   This caused AtomicFIFO and 4 other gates to be missed entirely.

### Pattern additions

- **x/exp/maps to stdlib migration** -- `maps.Keys` returns `iter.Seq[K]`
  in stdlib (not `[]K`), needs `slices.Collect` wrapper. Agents were
  incorrectly disabling the `inline` govet analyzer instead of migrating.
- **AtomicFIFO two-phase disable** -- `StaleControllerConsistency*`
  dependents must be disabled first, but ONLY in packages that import
  `k8s.io/kubernetes/pkg/features`. Env-var packages just need `os.Setenv`.
- **GetRawBytes** returns `[]byte` (no error) -- agents added wrong error
  handling when the pattern doc said `GetRawBytes`/`SetRawBytes`.
- **nilness dead code** -- stricter Go 1.26 govet catches dead `if err != nil`
  blocks after functions that don't return errors.
- **Test file type checking** -- `go build` doesn't compile `_test.go` files,
  so type divergence errors only surface during `go test` or `go vet`.
- **Mock regeneration** -- codegen can delete mock files; check `.mockery.yaml`.

### Infrastructure fixes

- **Validation timeout** -- increased from 15m to 25m (pkg/ovn has 518 tests)
- **Test fallback timeout** -- was hardcoded 10m, now uses VALIDATION_TIMEOUT
- **Anti-disable guidance** -- explicit "NEVER disable lint analyzers" rule

## What the Rebase Produces (k8s 1.35 to 1.36)

A successful rebase creates 14-18 commits covering:

1. Module rebases (conformance, e2e, go-controller)
2. go.mod sync for replace directives
3. Codegen (initially fails, then fix --bounding-dirs + regenerate)
4. Version reference updates (CI, docs, Go version, golangci-lint)
5. network-policy-api v0.2.0 type changes (WithConditions builder,
   peer type split, conformance rename, test structs)
6. Go 1.26 vet format string fixes (~12 files)
7. x/exp/maps to stdlib migration (~8 files)
8. Deprecated API fixes (reflect.Ptr, constraints.Ordered, FieldsV1.Raw,
   AddToScheme, nilness dead code)
9. AtomicFIFO feature gate disable (test-go.sh + 6-11 suite_test.go files)
