# Deep Dive: KubeEdge, Emissary-Ingress, Telepresence

File-level deep dives. Telepresence completed. KubeEdge and
Emissary-Ingress results pending.

## Telepresence — Key Findings

### 1. Go Dependency Submission API

File: `.github/workflows/go-dependency-submission.yaml`

Feeds Go dependency graph to GitHub's Dependency Graph API via
`actions/go-dependency-submission`. Points at specific build
target (`cmd/telepresence/main.go`). Enables Dependabot alerts
and dependency review for Go projects with custom build targets.

**MCN**: Easy to add. Enables GitHub's dependency features.

### 2. One-Shot Label Test Gating

File: `.github/workflows/dev.yaml`

Integration tests trigger on `labeled` event. First step
**removes** the label immediately. Adding the label again
re-triggers. Two labels select different test paths:

- `ok to test` — normal integration tests
- `compatibility test` — backward/forward against pinned version

**MCN**: Re-triggerable one-shot pattern. Cleaner than persistent
labels for expensive tests.

### 3. Content-Hash Dirty Version

File: `build-aux/genversion/main.go`

When `git status` shows uncommitted changes, computes MD5 over
all non-test, non-gitignored files. Base64-encoded with semver-
compatible character substitution. Same dirty tree always produces
same version, any change produces different version.

**MCN**: More deterministic than `+dirty` suffix. Useful for
debugging "which build am I running?" questions.

### 4. Sub-Component Release Tagging

File: `.github/workflows/release-teleroute.yaml`

Monorepo with separate Go modules. Main release creates two tags:
`vX.Y.Z` (main binary) and `rpc/vX.Y.Z` (RPC module). Teleroute
sub-component uses `teleroute-X.Y.Z` tag namespace (no `v` prefix).

**MCN**: If MCN has multiple Go modules, use separate tag
namespaces per module.

### 5. License Generation with Dependabot Auto-Commit

File: `.github/workflows/licenses.yaml`

`go-mkopensource` generates `DEPENDENCIES.md` and
`DEPENDENCY_LICENSES.md`. If Dependabot triggered the change,
auto-commits updated license files. Otherwise fails CI if drift
detected. Base64-encoded diff passed between jobs.

### 6. depguard Banning Internal Package Alternatives

File: `.golangci.yml`

Three lint tools enforce coding standards:

- `depguard`: bans `go-shellquote` -> use `pkg/shellquote`
- `gomodguard`: bans `gotest.tools` -> use `testify/go-cmp`
- `forbidigo`: bans `os.IsNotExist` -> use `errors.Is(fs.ErrNotExist)`

Most layered lint enforcement found.

### 7. macOS Signing as Optional Post-Release Step

File: `.github/workflows/release.yaml`

`environment: macos-signing` requires manual approval. If nobody
approves, release proceeds without signed macOS `.pkg`. Signed
packages uploaded later via `gh release upload --clobber`.

**MCN**: "Release now, sign later" pattern with Environment gate.

### 8. Pin-File Pattern for Tool Versions

Each tool under `tools/src/` has a `pin.go` file with
`//go:build pin` constraint and its own `go.mod`. Makefile generic
pattern extracts import path via sed and builds. Avoids single
`tools.go` anti-pattern.

**MCN**: Cleaner tool isolation than the `tools.go` approach.

### 9. GOEXPERIMENT=jsonv2 Globally

Makefile and CI both set `export GOEXPERIMENT := jsonv2`. First
project surveyed adopting an experimental Go feature project-wide.

### 10. CHANGELOG.yml with Auto-Date Stamping

YAML changelog (not markdown). `sed` replaces `date: (TBD)` with
current date on GA releases. Custom Go tools generate `.md`, `.mdx`,
and `variables.yml` outputs from the structured YAML.

**MCN**: YAML-based changelog is more machine-parseable than
markdown.

## Emissary-Ingress — Key Findings

### 11. Python-Generated GoReleaser Config

File: `make-gorel.py`

GoReleaser YAML generated by Python script with class-based
abstractions for 7 components. Handwritten prologue + generated
body. First repo where GoReleaser config is a generated artifact.

**MCN**: If GoReleaser config gets complex, consider generating
it from a higher-level definition.

### 12. Make Variable-Value Dependency Tracking

File: `build-aux/var.mk`

`$(var.)FOO` as a Make dependency causes rebuild when variable
value changes between runs. Uses hidden files with write-if-changed.
Make-level reactive system for parameterized builds.

**MCN**: Elegant but complex. Study if MCN Makefile has
parameterized build targets.

### 13. nektos/act Support in Production Workflows

Workflows detect `${{ env.ACT }}` and adjust: install missing
tools, reduce architecture list to native only, alternate trigger
paths. First-class local CI runner support.

**MCN**: If developers want to run CI locally, bake act support
into workflows.

### 14. uv as Primary Python Toolchain

Full `uv` adoption: `uv sync`, `uv run`, `uv venv`, `uv pip`,
even `uv python install`. Among earliest large CNCF projects to
fully adopt uv over pip/virtualenv.

**MCN**: Use `uv` for any Python tooling in CI (faster than pip).

## KubeEdge — Key Findings

### 15. Doc-Skip Companion Workflows

Files: `.github/workflows/cifuzz-doc.yaml`,
`fossa-doc.yaml`, `main-doc.yaml`

Paired workflow files with inverted path filters satisfy required
status checks for doc-only PRs. Simpler than `paths-ignore` when
you have many required checks.

### 16. Split DaemonSet Pattern for Cloud/Edge Cilium

File: `hack/cilium_e2e_test.sh`

Deploys two Cilium DaemonSets: one for cloud nodes (excluding edge
via node selector), one for edge nodes pointing at edge API server.
Label-gated E2E via `cilium-e2e-test` label.

**MCN**: Reference architecture for MCN's potential edge computing
integration with Cilium.

### 17. SLSA Dual Provenance (Binaries + 12 Container Images)

File: `.github/workflows/release.yml`

Binary provenance combines hashes from 11-output matrix job.
Container provenance uses dynamic output reference per image.

### 18. UPX Binary Compression for Edge

File: Makefile (`smallbuild` target)

`upx-ucl -9` after static linking produces minimal binaries for
resource-constrained edge devices.

**MCN**: Consider for MCN agents running on edge nodes.
